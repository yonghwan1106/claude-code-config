# 경인블루저널 기사 작성

## 매체 정보

- **매체명**: 경인블루저널 (인터넷언론사)
- **기자**: 박용환 기자
- **바이라인**: `경인블루저널 박용환 기자`

## 워크플로우

### 1단계: 기사 유형 판별

사용자 요청에서 기사 유형을 판별한다.

| 단서 | 유형 |
|------|------|
| 팩트 전달, 발표, 보도자료 | 스트레이트 |
| 심층 분석, 원인, 대안, 시리즈 | 기획 |
| 대화, 질의응답, 인물 | 인터뷰 |
| 주장, 의견, 제언 | 칼럼/사설 |
| 현장 방문, 취재, 목격 | 르포 |
| 사건, 사고, 수사 | 사건사고 |

판별이 불확실하면 사용자에게 확인한다. 유형별 상세 구조는 [references/article-types.md](references/article-types.md) 참조.

### 2단계: 입력 분석

**주제만 제공된 경우:**
- 핵심 팩트 확인 질문 (육하원칙 기반)
- 충분하면 바로 초안 작성

**자료가 함께 제공된 경우:**
- 보도자료/팩트시트/회의록에서 핵심 팩트 추출
- 인용 가능한 발언 식별
- 수치/데이터 정리
- 추가 확인이 필요한 사항 메모

### 3단계: 기사 작성

아래 순서로 작성한다:

1. **제목** — SEO 최적화, 20~35자
2. **부제** — 제목 보완 (스트레이트·기획 필수, 나머지 선택)
3. **리드** — 1~2문장, 핵심 사실 집약
4. **본문** — 유형별 구조에 따라 전개
5. **바이라인** — `경인블루저널 박용환 기자`
6. **사진 캡션** — 사진 정보 있으면 캡션 제안, 없으면 "추천 사진 + 캡션" 제안
7. **SEO 메타** — 태그 5~8개, 메타 디스크립션 80~120자

SEO 상세 가이드: [references/seo-guide.md](references/seo-guide.md) 참조.

### 4단계: 출력

TXT 파일로 저장한다. 파일명 형식: `YYYYMMDD_기사제목_요약.txt`

```
[제목]
기사 제목

[부제]
부제 (있는 경우)

[리드]
리드 문단

[본문]
본문 내용...

[사진 캡션]
1. 캡션 내용 ⓒ출처

[바이라인]
경인블루저널 박용환 기자

---
[SEO 메타]
태그: 태그1, 태그2, 태그3, ...
메타 디스크립션: ...
```

## 문체 규칙

### 공통
- 경어체 금지 (해요체 X) — `~다`, `~했다` 종결
- 한 문장 40자 이내 권장
- 불필요한 수식어 배제
- 외래어 표기법 준수
- 숫자는 아라비아 숫자 (`3명`, `50%`)
- 따옴표: 직접인용 `"큰따옴표"`, 강조/별칭 `'작은따옴표'`

### 기사 유형별 톤
- **스트레이트/사건사고**: 건조·객관적, 과거형 중심
- **기획**: 분석적, 현재형+과거형
- **인터뷰**: 대화체 살리되 정제, 직접인용 적극 활용
- **칼럼**: 설득적, 1인칭 허용
- **르포**: 묘사적·감각적, 현재형 중심

## 리라이팅/교정 모드

사용자가 기존 기사의 교정 또는 리라이팅을 요청하면:

1. 원문을 먼저 읽는다
2. 문제점을 간단히 정리한다 (문체, 구조, 팩트, SEO 등)
3. 수정본을 작성한다
4. 원문 대비 변경사항을 요약한다

교정 범위:
- **어법/맞춤법**: 띄어쓰기, 조사, 어미
- **문체**: 기사체에 맞지 않는 표현 수정
- **구조**: 역피라미드/논리 흐름 재배치
- **SEO**: 제목·키워드 최적화 제안
